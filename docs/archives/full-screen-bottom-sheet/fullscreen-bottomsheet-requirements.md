# 테스트 케이스

## TC-R1. DisableDragArea 드래그 방지

- **TC-R1-1** DisableDragArea 내부(폼 요소, 스크롤 영역 등)에서 드래그해도 시트가 닫히지 않는다
- **TC-R1-2** DisableDragArea 내부의 스크롤·입력은 정상 동작한다
- **TC-R1-3** DisableDragArea 바깥(AppBar, 빈 공간)에서는 정상 드래그 닫기가 된다

## TC-R2. 전체 오버플로우 시 스크롤 + 드래그 닫기 자동 비활성화

- **TC-R2-1** 콘텐츠가 시트 높이를 넘으면 시트 본문이 스크롤되고 드래그 닫기가 비활성화된다
- **TC-R2-2** 콘텐츠가 동적으로 추가되어 overflow가 발생하면 즉시 드래그가 차단된다
- **TC-R2-3** 콘텐츠가 동적으로 제거되어 overflow가 해소되면 드래그 닫기가 다시 동작한다
- **TC-R2-4** AppBar는 오버플로우 상태와 무관하게 드래그 닫기가 된다

## TC-R3. 짧은 콘텐츠의 빈 공간 드래그 닫기

- **TC-R3-1** 콘텐츠가 짧을 때 시트 어디서든 dismiss 임계 거리 이상 드래그하면 닫힌다
- **TC-R3-2** dismiss 임계 거리 미만 드래그 후 놓으면 시트가 원래 위치로 스냅백된다
- **TC-R3-3** dismiss 임계 속도 이상의 빠른 스와이프는 거리 무관하게 닫힌다

## TC-R4. 키보드 열림 시 드래그 닫기 비활성화

- **TC-R4-1** 가상 키보드가 열리면 시트를 드래그해도 닫히지 않는다
- **TC-R4-2** 가상 키보드가 열려 있을 때 빠른 스와이프를 해도 닫히지 않는다
- **TC-R4-3** 키보드가 닫히면 드래그 닫기가 다시 동작한다
- **TC-R4-4** 키보드가 열린 상태에서 시트 내 스크롤은 정상 동작한다

## TC-R5. Pull-to-refresh 방지

- **TC-R5-1** 시트가 열린 상태에서 아래로 강하게 드래그해도 브라우저 pull-to-refresh가 트리거되지 않는다
- **TC-R5-2** 전체 스크롤이 맨 위인 상태에서 추가 아래 드래그 시에도 pull-to-refresh가 발생하지 않는다

## TC-A1. 드래그 dismiss 조건

- **TC-A1-1** dismiss 임계 거리 이상 드래그 후 릴리즈 → 닫힘
- **TC-A1-2** dismiss 임계 거리 미만 드래그 후 릴리즈 → 닫히지 않고 스냅백
- **TC-A1-3** dismiss 임계 속도 이상 스와이프 → 거리 무관하게 닫힘
- **TC-A1-4** dismiss 조건 충족 릴리즈 시 시트가 위로 살짝 튀어오르지 않는다 (dip-up 없음)
- **TC-A1-5** dismiss 조건 미달 릴리즈 시 원점으로 스프링 스냅백

## TC-A2. 열기/닫기 애니메이션

- **TC-A2-1** 열기: 시트가 아래에서 위로 슬라이드인된다
- **TC-A2-2** 닫기: 시트가 위에서 아래로 슬라이드아웃된다
- **TC-A2-3** 드래그 중: 손가락 위치에 따라 시트가 아래로 따라 내려간다
- **TC-A2-4** 드래그 복귀: 스프링 애니메이션으로 원래 위치로 돌아온다
