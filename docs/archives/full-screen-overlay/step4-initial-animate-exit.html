Step 4: initial / animate / exit 개념
framer-motion에서 세 가지 상태를 정의한다: 마운트 직후(initial), 목표 상태(animate), 언마운트 시(exit).
CSS transition으로 동일한 개념을 시뮬레이션한다.

Demo 1 — opacity 0 → 1 → (닫기) → 0

가장 기본. initial={{ opacity: 0 }} / animate={{ opacity: 1 }} / exit={{ opacity: 0 }}

initial
opacity 0 → 1
// framer-motion 코드 <motion.div initial={{ opacity: 0 }} animate={{ opacity: 1 }} exit={{ opacity: 0 }} />

Demo 2 — scale 0 → 1 → (닫기) → 0

기본 모드 (triggerRect 없을 때). initial={{ scale: 0 }}

initial
scale 0 → 1
// framer-motion 코드 <motion.div initial={{ scale: 0 }} animate={{ scale: 1 }} exit={{ scale: 0 }} />

Demo 3 — clip-path inset (triggerRect 모드, 실제 구현)

가운데 작은 rect에서 전체로 확장. 닫히면 다시 작은 rect로 오므라듦.

initial
clip-path: inset 애니메이션
// framer-motion 코드 (FullScreenOverlay 실제) <motion.div initial={{ clipPath: 'inset(버튼위치 round 16px)', opacity: 0 }} animate={{ clipPath: 'inset(0px 0px 0px 0px round 0px)', opacity: 1 }} exit={{ clipPath: 'inset(버튼위치 round 16px)', opacity: 0 }} />
