# 콘텐츠 배치 정책

MP와 인접 프로젝트(KA, AC) 사이에서 콘텐츠를 어디에 둘지 결정하는 기준이다.

다음 파일을 만지기 전에 반드시 읽는다:

- MP `docs/best-practices-map.md`
- MP `docs/patterns/<주제>.md`
- MP `docs/tips/<주제>.md`
- AC `deploy/contexts/coding-standards/<주제>.md`

## 분업 표

|           | KA (knowledge-archive)                  | MP patterns                                      | MP tips                                                | AC `deploy/contexts/coding-standards/`                                                                                                                                         |
| --------- | --------------------------------------- | ------------------------------------------------ | ------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 성격      | 개념·정의·원리, 배경·원문·방법론        | 판단 기준·규칙, 구현 패턴 (예시 코드 동반)       | 한 줄 결론·함정·체크포인트 (코드 예시는 한두 줄까지)   | 한두 줄 규칙으로 끝나는 것 (네이밍, 문법 선호, eslint 선호 등). 특정 기술스택에 종속되지 않는 범용 원칙·철학. 코드 예시가 붙더라도 예시는 설명 보조일 뿐 따라 쓸 템플릿이 아님 |
| 판단 질문 | "이게 뭐야?" / "왜?"                    | "이걸 어떻게 짜?" / "이렇게 짜야 한다"           | "이런 함정·결론은 기억해 둬야 함"                      | "공통 원칙·규약이 뭐야?"                                                                                                                                                       |
| 예시      | Array 정의, Random Access가 O(1)인 이유 | API 클라이언트 구조, 폼 처리 패턴, 오버레이 패턴 | "Vite는 `use client`를 strip하므로 보존 플러그인 필요" | 네이밍 컨벤션, eslint 선호 규칙                                                                                                                                                |

## MP 작성 규칙

`docs/patterns/`와 `docs/best-practices-map.md`는 AI가 코드를 쓸 때 참조하는 지침이다. 배경·원문·방법론은 KA로 위임한다.

### 패턴 문서에 넣지 않는 것

- 규칙의 근거·이론 설명 (why, false negative/positive 같은 논리)
- 1차 소스 URL 목록 (Fowler, Kent C. Dodds 등 외부 원문)
- 학습용 방법론 (예: "테스트 대상 결정 5단계" 같은 프로세스)
- 섹션마다 달리는 "출처: knowledge/..." 각주
- 배경 도입부, 연결 문장, 풍성한 설명
- 패턴 간 상호 참조 ("관련: TestWriting.md" 같은 링크 섹션) — `best-practices-map.md`가 전담

패턴 문서에는 판단 기준·규칙·MP 고유 코드/사례만 남긴다. 근거가 KA에 있으면 파일 맨 아래에 `배경: KA knowledge/<경로>` 한 줄로만 남긴다.

### tips 문서 형식

`docs/tips/<주제>.md`는 한 줄 결론·함정·체크포인트를 모은다.

- 한 항목 = 한 줄 결론 (필요 시 한두 줄 부연)
- 코드 예시는 한두 줄까지. 그 이상이면 patterns로 보낸다
- 배경·원문 인용·근거 설명은 KA로 분리한다
- 디렉토리는 첫 콘텐츠 추가 시점에 생성한다 (빈 자리는 두지 않는다)
